# SEO Optimization

## Meta tags & structured data

Handled centrally in `src/components/layout/BaseHead.astro`:

- Title tags & meta descriptions (unique per page)
- Open Graph & Twitter Cards
- `WebApplication` JSON-LD structured data
- Canonical URLs
- GA4 analytics scaffold (see below)

Several pages additionally carry their own `FAQPage` JSON-LD block
(`usage.astro`, `alternatives.astro`, `lorem-ipsum-api.astro`, and every
`lorem-ipsum-for/[slug]` page) for richer SERP presence.

## Sitemap

Auto-generated by `@astrojs/sitemap`, configured in `astro.config.mjs`.
Priority/changefreq is tiered by page type — see the `serialize()` function
there before adding a new page type, so it gets bucketed correctly:

- Homepage: 1.0, daily
- History / Usage: 0.8, monthly
- Alternatives / API / UI-design / Figma guides: 0.6, monthly
- `lorem-ipsum-for/*` (programmatic use-case pages): 0.5, monthly
- Legal pages: 0.3, yearly

## Domain configuration

Set in `astro.config.mjs`:

```javascript
export default defineConfig({
  site: "https://lorem-ipsum.bobadilla.tech",
  // ...
});
```

## Content strategy

Beyond the core generator, the site carries dedicated long-tail SEO pages:

- `/lorem-ipsum-api`, `/dummy-text-generator-ui`, `/placeholder-text-figma` —
  target specific developer/designer search intent.
- `/lorem-ipsum-for/*` — a programmatic set of use-case pages (landing pages,
  mobile apps, dashboards, etc.), each with genuinely distinct tips and a unique
  FAQ answer per page. Kept deliberately small (13 pages, not 100+) — thin
  near-duplicate templated pages are a doorway-page pattern search engines
  penalize, which would work against the goal. Add more via
  `src/lib/use-cases.ts`, but keep each entry's content genuinely distinct.

Internal linking (Footer "Guides" column, homepage "Explore More") points into
all of the above — check both when adding a new content page so it's actually
reachable by crawlers, not just present in the sitemap.

## Backlinks

Links to `requiems.xyz` (Requiems API) and `bobadilla.tech` (the agency) are
placed contextually — dev-tool CTAs on content/API pages, a footer credit line,
and a homepage "Developer API" section. Keep these pointed at the real live
domains; `requiems-api.xyz` does not resolve.

## After deployment

1. **Google Search Console** — verify domain, submit `sitemap-index.xml`.
2. **Bing Webmaster Tools** — same.
3. Confirm OG image renders in link previews (WhatsApp/Twitter/LinkedIn) after
   any change to `public/images/og-image.png` — social platforms don't render
   SVG previews, so it must stay a PNG at 1200×630.
